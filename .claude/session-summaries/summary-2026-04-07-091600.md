## Session Goal
Add attorney/trustee information extraction from county foreclosure notice PDFs. The "Attorney Info" column (col 10) in the Google Sheet was being filled with the **lender** name instead of actual attorney data. The user wanted the parser to extract the attorney/trustee firm or individual name from the PDFs and route it to the correct column. After implementing, the user asked to run the full pipeline end-to-end to verify results.

## Key Decisions Made
- **Extract attorney via regex patterns, same approach as other fields**: Added `ATTORNEY_PATTERNS` list to `parser.py` following the existing `_first_match()` pattern. Chose this over any ML/NLP approach because it's consistent with the codebase and the attorney info appears in predictable labeled sections of foreclosure notices.
- **18 regex patterns covering labeled fields + known firm names**: Patterns target "Substitute Trustee:", "Trustee:", "By:" signature blocks, "Attorney for Mortgagee:", and 12 hardcoded known firm names (Barrett Daffin, McCarthy & Holthus, Codilis & Moody, etc.). Known firms were derived from the existing `_ADDRESS_BLOCKLIST` comments.
- **Fall back to lender when attorney is empty**: In `sheets_exporter.py`, col 10 ("Attorney Info") now maps to `attorney or lender` — uses attorney when extracted, falls back to lender to preserve backward compatibility.
- **Added 6700 N. New Braunfels to address blocklist**: This attorney office address (San Antonio) was being incorrectly captured as a property address for the Fournier PDF. Adding it to `_ADDRESS_BLOCKLIST` fixed the false positive.
- **Boilerplate rejection in attorney patterns**: The first iteration incorrectly matched "under the deed of trust need not appear" as an attorney name. Fixed by adding negative lookaheads for boilerplate words (under, need, shall, will, may, is, the, to, of, or) after "Substitute Trustee:".
- **Pipeline state reset for fresh run**: Cleared `processed_pdfs` and `known_lead_keys` in `pipeline_state.json` and deleted old CSVs because the old `leads.csv` header didn't include the `attorney` column, and appending new rows with the column to an old header silently dropped the data.
