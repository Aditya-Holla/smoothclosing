## Key Decisions Made
- **Search relatives by name+state rather than clicking `<h6>` elements**: Clicking the relative's `<h6>` on the POI's results page triggers a search but then navigating back is unreliable in the React SPA. Searching independently by first name + last name + state "TX" costs the same 1 credit and avoids back-navigation issues.
- **Use `gender-guesser` PyPI package for relationship inference**: Determines likely gender from first names to distinguish Mother/Father, Brother/Sister, Son/Daughter, Husband/Wife.
- **Wait for "Property Details" text instead of `networkidle`**: Skip Genie's AJAX-based results don't trigger `networkidle` reliably. Changed to `page.wait_for_function()` that checks for "Property Details" or "No results" in body text.
- **Don't auto-fill login credentials**: The `_react_fill` function wasn't properly updating React state for the login form, causing "no account found" errors. Changed to fully manual login with 3-minute wait.
- **Skip phones for deceased relatives**: Per coworker's requirements, deceased persons get their address and deceased flag but no phone numbers in output.
- **Dedup cache for relative searches**: Module-level `_search_cache` dict prevents re-searching the same person across multiple leads, saving credits.

### Constraints Discovered
- **Skip Genie React forms**: Standard `page.fill()` doesn't trigger React's onChange. Must use native value setter + dispatching input/change events (`_react_fill()` helper).
- **Multiple "Get Info" and "Yes, Execute Search" buttons in DOM**: Skip Genie has ~11 hidden "Yes, Execute Search" buttons. Must use `:visible.first` selector.
- **AJAX results don't trigger networkidle**: The Property Details modal loads via AJAX after clicking "Yes, Execute Search" but `page.wait_for_load_state("networkidle")` resolves before the AJAX completes. Must use `page.wait_for_function()` checking for specific text.
- **reCAPTCHA on login**: Cannot be automated. User must solve manually. Persistent context saves session so it's only needed once.
- **Terms of Service popup**: Appears after login, can block the search page. The `_is_on_search_page()` function checks for blocking modals.
- **Relative name-only searches often return "No results"**: Common names without address context fail frequently on Skip Genie.
- **Deceased detection**: The word "Deceased" appears somewhere in Skip Genie result pages for deceased persons, but the exact location/format needs verification to avoid false positives.
