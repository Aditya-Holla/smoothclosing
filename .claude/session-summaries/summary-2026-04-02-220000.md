### Constraints Discovered
- **RingCentral JWT is extension-locked**: A JWT created by user X can only send SMS from numbers assigned to user X's extension. Chip's JWT can't send from Vince's number and vice versa
- **"Phone number doesn't belong to extension" (MSG-304)**: Returned when trying to send from a number on a different extension than the JWT owner
- **"Cannot find the phone number which belongs to user" (MSG-245)**: Returned when the number isn't assigned to the extension at all (e.g., wrong number)
- **Softphone numbers CAN send SMS**: Vince's softphone number (512) 996-3402 worked for sending even though it's not a "Direct Number"
- **RingCentral developer "Demonstration App" is read-only**: Can't edit its settings. Had to create a new app
- **JWT tokens can be accidentally concatenated**: When pasting Vince's JWT after Chip's in the .env, they merged into one invalid 5-part token. Valid JWTs have exactly 3 parts (header.payload.signature)
- **GitHub Actions runner has no Skip Genie session**: The .skipgenie_session/ directory is local-only. CAPTCHA must be solved on a machine with a headed browser
- **System Python 3.9 on Mac needed pip installs**: `python-dotenv` and `requests` weren't available by default
