## Session Goal
Transform the monolithic SmoothClosing foreclosure pipeline into an agentic architecture using the Claude Agent SDK. The user wants an orchestrator agent that team members (2-3 people) talk to in natural language, which delegates to specialized subagents wrapping the existing Python scripts. The immediate scope is the **acquisitions pipeline only** — dispositions and tax foreclosures will be added later as separate phases.

The broader vision: multiple teams (foreclosure acquisition, dispositions, future probate/tax liens) share Skip Genie and RingCentral SMS as common services, with a shared knowledge base (the Google Sheet) preventing double-contacts across teams.

## Key Decisions Made
- **Claude Agent SDK over pure Python agents**: User explicitly chose the Claude Agent SDK for the LLM-powered orchestration rather than a custom Python framework. This gives natural language routing, multi-step reasoning, and smart decision-making.
- **Orchestrator + subagents over single agent**: User chose the full multi-agent architecture (orchestrator delegates to 4 specialized subagents) rather than a simpler single-agent approach. Reasoning: wants clear separation of concerns and the ability to add more teams/agents later.
- **Google Sheet as shared state, no SQLite KB**: Originally planned a separate SQLite knowledge base for cross-team contact dedup. Decided against it because the Google Sheet already serves as the team's shared working document — adding a second source of truth would create drift. The Sheets Agent reads the Sheet to check contact history instead.
- **Bash-as-tools pattern**: Rather than writing MCP tool wrappers for each existing module, the subagents use the SDK's built-in Bash tool to invoke existing CLI scripts directly. All modules already have argparse CLIs, and CSVs serve as the data contract between stages. This avoids maintaining a second interface.
- **Sonnet 4.6 for all agents**: Chose claude-sonnet-4-6 for the orchestrator (routing + synthesis tasks). Subagents inherit the same model. Keeps costs reasonable for a small operation.
- **Python 3.12 venv**: Created a separate `.venv-agents/` venv using Anaconda's Python 3.12 since the Claude Agent SDK requires 3.10+ and the system Python is 3.9.
- **Acquisitions first, then expand**: User decided to fully automate acquisitions before building dispositions or tax foreclosure agents. The shared services (skip trace, SMS) are already designed to be team-agnostic.

## Technical Context
### Architecture / Stack
```
Team Member (CLI chat)
       │
  Orchestrator Agent (claude-sonnet-4-6)
  ├── Tools: Agent, Read, Glob
  │
  ├──→ PDF Agent [Bash, Read, Glob]
  │      county_downloader.py → main.py → equity_estimator.py
  │
  ├──→ Skip Trace Agent [Bash, Read]
  │      skipgenie.py (Playwright browser automation)
  │
  ├──→ SMS Agent [Bash, Read]
  │      ringcentral_sms.py (--dry-run safety)
  │
  └──→ Sheets Agent [Bash, Read]
         sheets_exporter.py (push + query)
```

- **Claude Agent SDK v0.1.56** — provides `query()`, `ClaudeSDKClient`, `AgentDefinition`, `ClaudeAgentOptions`
- **Subagent isolation**: Subagents start fresh with no parent conversation history. They receive only their own system prompt + the delegation prompt from the orchestrator.
- **Session resumption**: The CLI captures `session_id` from `ResultMessage` and passes it via `opts.resume` on subsequent queries for multi-turn context.
