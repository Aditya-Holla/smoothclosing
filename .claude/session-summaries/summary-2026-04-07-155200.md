## Key Decisions Made
- **Claude Agent SDK over custom framework**: User explicitly chose the Agent SDK for LLM-powered orchestration
- **Orchestrator + subagents over single agent**: User chose full multi-agent architecture despite being shown a simpler single-agent alternative. Reasoning: clear separation of concerns, ability to add more teams/agents later.
- **Google Sheet as shared state, no SQLite KB**: Originally planned a SQLite knowledge base. Dropped it because the Google Sheet already serves as the team's shared working doc — adding a second source of truth would create drift.
- **Bash-as-tools pattern**: Subagents invoke existing CLI scripts via the SDK's built-in Bash tool rather than MCP wrappers. All modules already have argparse CLIs.
- **No OpenCorporates / LLC lookup**: Explored OpenCorporates (needs paid API key ~$2800/yr), Texas SOS ($1/search), Cobalt Intelligence ($1000/mo). All too expensive for <50 lookups. User decided to skip LLC officer lookup and just skip trace buyer names directly.
- **Name-only skip tracing**: Updated skip-trace-agent to support `skip_genie_search.py` (Mode 2) for name-only searches without addresses, and `buyer_tracer.py` (Mode 3) for Sheet-to-Sheet buyer tracing.
- **Simple Texting for dispositions, RingCentral for acquisitions**: Both texting services coexist. Simple Texting handles inbound auto-responses for the dispositions team.
- **Text only, no calls**: User specified all responses should suggest texting, never calling. "We'll go over the specifics later."
- **Dispositions sheet separate from acquisitions**: User wanted a brand new Google Sheet for dispositions with metro tabs (Austin, Houston, San Antonio, Dallas).
- **permissionMode="acceptEdits"**: All subagents need this to auto-approve Bash/Read/Write tool use without interactive confirmation. Without it, agents hang waiting for approval.

### Constraints Discovered
- **Claude Agent SDK requires Python 3.10+**: System Python is 3.9.6, separate venv needed
- **Subagent isolation**: Subagents don't see orchestrator's conversation. Context must be passed in delegation prompt.
- **permissionMode="acceptEdits" required**: Without this, subagents hang waiting for interactive tool approval
- **BIS CAD sites use Blazor Server-Side rendering**: Results loaded via SignalR websocket, not in initial HTML. Must use Playwright and wait for rendering.
- **Bell/Burnet use reCAPTCHA Enterprise**: Cannot be solved in Playwright (detects automation). Standard reCAPTCHA checkbox click doesn't work.
- **Travis CAD uses Prodigy platform (React)**: Different from BIS, needs its own search + result parsing logic
- **Skip Genie name search requires State field**: Without filling "TX" in the state input, searches return no results
- **BIS address search needs tab switching**: Keywords search only matches owner names. Address search requires clicking "By Address" tab and splitting address into StreetNumber + StreetName fields.
- **buyer_tracer.py address parsing**: Skip Genie returns addresses like "1148 RUSTY BLACKHAW TRL SAN MARCOS TX 78666 HAYS" (no commas, county after zip). Custom parser splits on street suffix + directional + state + zip pattern.
