## Session Goal
Implement the coworker's address-first Skip Genie search flow with updated relationship inference algorithm and h6 relative clicking.

## Key Decisions Made
- **Address-first search**: Search by property address first, then look for the owner among results. If owner not found, fall back to name-only search (no address fields).
- **Owner not found = manual_review**: When address returns results but owner isn't among them AND name search fails, mark as `manual_review`.
- **H6 clicking for relatives**: Click the relative's `<h6>` element directly instead of searching by name+state. Costs the same 1 credit but is more accurate.
- **Back-navigation fallback chain**: Escape → go_back() → re-execute address search. The re-search fallback is always used since Escape/go_back don't work reliably in the React SPA. Credits are unlimited so this is fine.
- **Skip address search for no-address leads**: When there's no property address, go straight to name-only search instead of timing out.
- **Deceased POI = no phones**: Per coworker's rules, deceased owners get their address and deceased flag but no phone numbers.
- **Address Search tab**: The tab is an `<li>` element (`li.tabs:has-text("Address")`), not a `<button>`. The Get Info button must use `:visible` selector since there are two (one per search form).
